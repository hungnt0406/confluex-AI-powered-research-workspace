Stages
Stage-by-Stage Journey
1 Discover & Decide to Try Planned
GoalFind a faster alternative to 2-4 months of manual literature review.
ActionsLands on the app, reads pitch, clicks "Try it."
SystemPublic landing page + live demo URL (Phase 4/5).
FeelingSkeptical but curious
2 Sign Up / Log In Shipped
GoalGet access quickly with minimal friction.
ActionsRegister with email/password, or log in.
SystemPOST /auth/register, POST /auth/login → JWT token.
FeelingNeutral
3 Create Project & Define Topic Shipped
GoalTranslate a vague research idea into a runnable project.
ActionsEnter title, topic description, year range, candidate/summary limits.
SystemPOST /projects persists project defaults. First chat message creates project.
FeelingUncertain
4 Upload Seed Reference PDFs Shipped
GoalBring existing reading into the system so results are anchored.
ActionsDrag-and-drop PDFs into the project composer.
SystemPOST /projects/{id}/reference-files → validate, store, parse, create linked Paper.
FeelingReassured
5 Run Discovery Pipeline Partial
GoalTurn the topic into a ranked, summarized paper shortlist (~90s).
ActionsClicks "Run" — Searcher expands queries, Reader ranks + summarizes.
SystemPOST /projects/{id}/run → LangGraph discovery graph (searcher → reader).
FeelingAnxious + excited
GapNo SSE progress stream yet.
6 Browse Ranked Paper List Shipped
GoalQuickly decide which papers matter — triage 30 papers in minutes.
ActionsSort by relevance, filter, expand summaries, view citation graphs, multi-select.
SystemGET /projects/{id}/papers with filtering, pagination, joined citation counts.
FeelingEmpowered
7 Deep-Dive Paper Q&A Shipped
GoalUnderstand papers without reading every PDF end-to-end.
ActionsSelect 0-5 papers, ask grounded questions, follow up in persisted conversations.
SystemProject-scoped chat with streaming SSE, on-demand PDF extraction, top-k retrieval.
FeelingConfident
8 Select Papers & Request Writing Partial
GoalGet a ready-to-paste writing artifact grounded in chosen papers.
ActionsMulti-select papers, enter instruction, choose output + citation + reference format.
SystemPOST /projects/{id}/writer/generate → grounded Writer agent + QA validator.
FeelingIn control
GapWriter workspace UI not yet exposed in frontend.
9 Review Output + QA Flags Partial
GoalConfirm the generated artifact is academically defensible.
ActionsRead body, inspect citations, review QA flags with severity and location.
SystemQA enforces: unsupported claims, missing references, malformed LaTeX, incomplete metadata.
FeelingCritical → Trusted
GapOutput review UI still planned; backend/API only.
10 Iterate / Regenerate Partial
GoalImprove the artifact without starting from scratch.
ActionsAdjust instruction, swap papers, change format, regenerate.
SystemNew writer requests produce new persisted outputs; prior outputs remain accessible.
FeelingProductive
GapNo diff view or partial regeneration yet.
11 Copy / Export Planned
GoalTake the artifact into LaTeX, Google Docs, Word, or Markdown.
ActionsCopy body + references, or download in chosen format.
SystemDeterministic formatter emits LaTeX \cite{} + BibTeX, numbered [1], or author-year.
FeelingSatisfied
Gap.bib, .tex, .docx download endpoints not yet implemented.
12 Return & Manage Library Partial
GoalRun more projects, revisit prior outputs, keep an organized research library.
ActionsView project list, revisit outputs, upload more PDFs, rename/delete projects.
SystemGET /projects, PATCH, DELETE, per-project library, persisted conversations + outputs.
FeelingInvested / Loyal
GapNo tagging, annotations, or cross-project search yet.
Key Risks
Critical Moments of Truth
Stage 5 — First 10 seconds
Discovery must feel alive with progress feedback and SSE streaming.
Stage 6 — Paper list quality
If the top 5 papers look irrelevant, the user churns immediately.
Stage 7 — First grounded answer
Must reference the actual paper text, not a generic abstract restatement.
Stage 8 — First writer output
Must obey the instruction exactly, citing only selected papers.
Stage 9 — QA-flag clarity
Each flag must explain why and be actionable — the demo-defining moment.
Stage 11 — Copy fidelity
\cite{} keys and BibTeX must paste cleanly into Overleaf / Word without fixes.
